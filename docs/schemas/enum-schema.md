# Enum Schema

Enum schemas define the valid values for enum properties and scope parameters.

## Schema Directory

The default factory loads enum schemas from JSON files in the `schemas/enums/` directory at the root of the project. This can be changed in `config/entity-requester.php`:

```php
'enum_schema_directory' => base_path('custom/path/to/enums'),
```

This setting is only used by the default factory. If you use a custom factory (see below), it is responsible for loading schemas on its own.

## Custom Factory

You can replace the built-in factory with your own implementation (e.g., to load schemas from a database or an API). Create a class that implements `EnumSchemaFactoryInterface` and bind it in a service provider:

```php
use Comhon\EntityRequester\Interfaces\EnumSchemaFactoryInterface;

$this->app->singleton(EnumSchemaFactoryInterface::class, MyEnumSchemaFactory::class);
```

If your custom factory supports caching, implement `CacheableInterface` as well to enable cache refresh via `EntityRequester::refreshEnumCache()`.

## Structure

```json
{
  "id": "fruit",
  "name": "fruit",
  "type": "string",
  "cases": [
    { "id": "apple", "name": "apple" },
    { "id": "orange", "name": "orange" },
    { "id": "strawberry", "name": "strawberry" }
  ]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string` | Unique identifier for the enum |
| `name` | `string` | Human-readable name |
| `type` | `string` | Backing type: `string` or `integer` |
| `cases` | `array` | List of valid enum values |

Each case has:

| Field | Type | Description |
|-------|------|-------------|
| `id` | `string\|int` | The value (matches PHP enum's backed value) |
| `name` | `string` | Human-readable label |

## Usage in Entity Schema

**Property with enum:**
```json
{ "id": "status", "type": "integer", "enum": "status", "nullable": true }
```

**Array of enums:**
```json
{
  "id": "favorite_fruits",
  "type": "array",
  "items": { "type": "string", "enum": "fruit" },
  "nullable": true
}
```

**Scope parameter with enum:**
```json
{
  "id": "fruit",
  "name": "fruit",
  "type": "string",
  "enum": "fruit",
  "nullable": false
}
```

## Auto-Generation

Enum schemas are automatically generated by the `entity-requester:make-model-schema` command when it detects PHP backed enums in model casts or scope parameters. The enum must be a backed enum (`string` or `int`).

```php
enum Fruit: string
{
    case Apple = 'apple';
    case Orange = 'orange';
    case Strawberry = 'strawberry';
}
```

Generates:

```json
{
  "id": "fruit",
  "name": "fruit",
  "type": "string",
  "cases": [
    { "id": "apple", "name": "apple" },
    { "id": "orange", "name": "orange" },
    { "id": "strawberry", "name": "strawberry" }
  ]
}
```
